01Interaction model

The load-bearing rules that govern every screen. If you only read one section, read this.
Depth is the core gesture. Vertical swipe on the home canvas cycles Glance → Scan → Dive. Depth persists per-surface (home, radar, alerts) and survives app restart. Default is Scan, chosen during onboarding.
Back is always a chevron in the top-left. No gesture-only back. Detail screens always animate in from the right; sheets animate up from the bottom.
Life-safety is non-negotiable. Tornado Emergency, Flash Flood Emergency, and any CAP event flagged Extreme + Immediate bypass quiet hours, silent mode, and current screen. They render as full-screen takeovers that require an explicit acknowledgement tap.

02Launch & onboarding

First-run only. After completion, the user skips directly to Home on every subsequent launch. State is stored in secure device storage.

W-01WelcomeLaunch

Purpose
Set the product's voice and explain what Skybureau is and isn't. One paragraph, max.
Tap targets
Get started→ W-02Primary CTA
I already have an account→ sheetRestore prior settings via iCloud / Google Sign-In
Notes
No account is required. The app works fully offline-first with on-device location.

W-02PermissionsLaunch

Purpose
Request precise location and push notifications. Both are optional but strongly recommended.
Tap targets
Allow location→ systemTriggers iOS/Android permission dialog
Allow notifications→ systemTriggers system permission dialog
Continue→ W-03Proceeds even if denied
Skip→ W-04User can enter location manually
Denied state
If location denied, onboarding routes to W-04 (search) with a banner explaining degraded functionality.

W-03Depth choiceLaunch

Purpose
User picks a default information density. Reversible later in Settings.
Tap targets
Glance→ W-04Sets default depth; minimal data
Scan (recommended)→ W-04Sets default depth; balanced
Dive→ W-04Sets default depth; dense / pro

W-04Location searchLaunch

Purpose
Set home location. If location permission was granted, the current city is preselected.
Tap targets
Search fieldinlineAutocomplete via Mapbox / Apple MapKit
Use current locationinlineVisible only if location granted
Result row→ H-01Sets home, closes onboarding

W-05Notification priming (post-first-forecast)Launch

Purpose
If the user skipped push permission at W-02, re-prompt contextually after they see their first forecast. Stop asking after two declines.
Trigger
On fourth app open or when first significant alert is available for home location, whichever comes first.

04Depth system

Three information densities on the same forecast data. Gesture-driven, not menu-driven.

D-01GlanceDepth

Contents
Big temperature number, one-sentence natural-language summary, single hero icon, any active alert banner. Nothing else.
Intent
Morning-glance use case. Read in under 1 second.

D-02ScanDepth

Contents
Hero temperature, 12-hour chip strip, 7-day mini, radar preview, AQI + UV pair, alerts. This is the default.

D-03DiveDepth

Contents
Everything in Scan plus wind/pressure/humidity trend sparklines, model confidence bands, 10-day, and (when a convective watch is active) SPC outlook inset.
Notes
Dive surfaces provenance pills — every data card shows ◇ source · lead time · update time.

05Alert system

From subtle banner to full-screen takeover. The escalation is controlled by the NWS CAP event severity + urgency fields, mapped to our five-level scale.

A-01Ambient statementAlert

Trigger
CAP event severity = Minor, urgency = Future.
Treatment
Inline text below hero; no color accent. Not a tap target.

A-02Advisory bannerAlert

Trigger
CAP severity = Moderate.
Treatment
Yellow pill at top of home. Tap opens A-05.

A-03Watch bannerAlert

Trigger
CAP severity = Severe, urgency = Expected.
Treatment
Orange banner. Triggers push if enabled.

A-04Warning bannerAlert

Trigger
CAP severity = Severe, urgency = Immediate.
Treatment
Red banner, haptic on appearance. Push. Long-press to dismiss (session only).

A-05Single alert detailAlert

Contents
Full CAP text (headline, description, instruction), affected area polygon on map, expiration countdown, Safety drawer link.
Tap targets
What to do→ A-07Safety drawer
Share→ systemShare sheet; deep-link to alert
View on map→ F-03Full radar with polygon overlay

A-06Emergency takeoverAlert

Trigger
CAP severity = Extreme, urgency = Immediate. (Tornado Emergency, Flash Flood Emergency, specific civil emergencies.)
Treatment
Full-screen magenta takeover. Bypasses silent mode and quiet hours. Requires explicit acknowledgement tap to dismiss; cannot be swipe-dismissed.
Notes
Logs acknowledgement with timestamp. Useful for testing by support.

A-07Safety drawerAlert

Contents
Plain-language what-to-do for this specific hazard type. Sourced from Ready.gov + NWS safety pages; curated in-repo, not scraped at runtime.

06Forecast detail

Tapping a card on Home opens its dedicated detail screen. All detail screens support depth — Dive adds model confidence and provenance.

F-01Hourly detailDepth

Contents
48-hour temperature + precip + wind strip. Minute-cast inset for next 60 min when precip is near.

F-0210-day forecastDepth

Contents
Day rows with hi/lo, precip chance, icon, confidence band (Dive only).

F-03Radar · fullDepth

Purpose
One radar screen, three user-chosen styles. Timeline, layers, and data are identical; overlay density and vocabulary differ. Style is picked in S-06 and persists.
Shared chrome
Full-bleed map, −2h → now → +90m timeline with play/scrub, layer toggle (reflectivity / velocity / satellite / temperature / lightning), alert polygons overlaid when active, location pin.
Style · Clean (default)
Reflectivity tiles + timeline only. No annotations. For users who know how to read radar and don't want a cluttered view.
Style · Enthusiast
Adds storm-cell tracking (rotation markers, hail cores, cell IDs, projected tracks with arrival-time ticks), lightning strikes, storm-relative motion vectors. Tap a cell → F-03b.
Style · Narrative
Plain-language overlays that sit on top of the map: "Light rain reaches you in 18 min", "Storm clearing by 7:45 PM", "No precipitation expected in the next 90 minutes." For non-enthusiasts who still want radar on the home screen.
Tap targets
Storm cell (Enthusiast only)→ F-03bCell detail sheet
Layer chipinlineCycles layer
TimelineinlineScrub to any time in window
Style label (top-right)→ S-06Switch style

F-03bStorm-cell detail (sheet)Depth

Trigger
Tap any tracked cell on Enthusiast radar.
Contents
Cell ID, VIL, echo-top height, rotation (shear), hail probability + size, projected track with arrival-time ticks for up to 3 downstream locations, affiliated watch/warning polygons. Hazard chips (tornado / hail / wind / lightning) pill up when thresholds are crossed.
Sheet behavior
Half-height bottom sheet, draggable to full. Dismiss taps outside or chevron-down.

F-04Wind · DiveDepth

Contents
Speed + gust sparklines, compass rose, pressure trend, ensemble spread.

07Settings tree

Settings index is a list of categories. Each row pushes a focused detail screen. No nested sub-menus more than one level deep.

S-01Settings · indexSettings

Rows
Depth default · Units · Locations · Alert categories · Notification timing · Brief personalization · Accessibility · Specialty opt-in · Data sources · Location privacy · Export my data.

S-02Alert categoriesSettings

Contents
12 NWS product types grouped by hazard class. Life-safety row is locked ON and shows a locked pill.

S-03Notification timingSettings

Contents
Morning brief time, quiet hours window, precip nowcast lead time, routine windows (commute, dog walk).
Data write
Every change posts settings.updated event to the notification scheduler.

S-04Brief personalizationSettings

Contents
Coverage toggles, commute route + mode, wardrobe thresholds (sliders), sensitivities (pollen, AQI, migraine pressure), tone + reading level.

S-05AccessibilitySettings

Contents
Text size (6 stops), bold weight, line spacing, high contrast, reduce transparency, color-blind palette picker, reduce motion, audio playback rate, verbose screen-reader labels.

S-06Radar styleSettings

Purpose
Picks which style F-03 renders in. Persists per device.
Contents
Three large option cards — Clean, Enthusiast, Narrative — each with a 2-line description, a miniature preview, and an example sentence (for Narrative) or a labeled annotation (for Enthusiast). A footnote explains that data and layers are identical across styles; only the annotation layer changes.
Default
Clean. Depth default from onboarding (W-03) nudges this: Dive users default to Enthusiast, Glance users to Narrative.

08Agentic · AI

Three careful AI moves. Every output shows sources; none fabricate numbers. All generation happens server-side with deterministic fallbacks.

AI-01Skybureau BriefAI

Trigger
Daily at user's configured brief time (default 6:45 AM local). Also surfaced as a home card between 5–10 AM.
Contents
Templated paragraphs filled with real forecast values. Sections shown are controlled by S-04 toggles.
Fallback
If generation fails, render a structured template with raw forecast values — never show an empty or hallucinated brief.

AI-02Ask SkybureauAI

Purpose
Conversational Q&A scoped strictly to weather, location, and forecast provenance. Refuses unrelated queries.
Transparency
Every answer includes a source pill (NWS · ECMWF · HRRR · etc.) and the timestamp of the underlying data.

AI-03Minute-castAI

Purpose
60-minute precipitation nowcast with confidence bands. Surfaced automatically when precip is starting / ending within 90 minutes.
Data
MRMS + HRRR blend; confidence band rendered from ensemble spread.

12Accessibility targets

Public-service product: must meet WCAG 2.2 AA at minimum, with explicit additional commitments below.
Text contrast
All text meets 4.5:1 at Default size; 3:1 for large text. High-contrast mode pushes this to 7:1.
Text sizing
Six stops (XS–XXL) plus respect for system Dynamic Type / fontScale. No text is baked into SVG — all labels are real DOM nodes.
Color-blind
Severity scale has a non-color redundancy (pattern/icon) on alert banners. Three alt palettes ship (Deuter/Prota/Tritan).
Motion
Reduce-motion stills ambient sky motion, plays radar at 1× only, and disables Depth-cycle gesture animation.
Screen reader
Every numeric reading is announced with its unit. Radar frames have spatial-narration alt text ("storm cell moving northeast at 35 mph, 12 miles west of you").
Hit targets
Minimum 44×44 pt on iOS, 48×48 dp on Android.
Audio
All brief and alert copy is available as read-aloud using the system voice; user can tune speaking rate (0.8–1.5×).
